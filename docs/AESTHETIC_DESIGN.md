# 天空织机 · Skyloom — 美学设计系统

> 设计哲学：**水墨气象台 (Ink-Wash Weather Station)**。
> 不是极简，是**克制的繁复**——东方水墨的留白与西方信息设计的精确并置。
> 每个 Agent 是一种天气、一味矿物颜料、一句诗、一种动势。辨识度来自这套贯穿 CLI / TUI / Web 的统一意象。

---

## 0. 设计立场

| 反对 | 主张 |
|------|------|
| 扁平、纯灰、性冷淡的"AI 默认风" | 宣纸质感、矿物颜料、笔触与晕染 |
| 装饰为零的极简 | **有意义的繁复**：每个元素承载身份（天气/颜料/诗/动势） |
| 千篇一律的等宽终端 | 真彩矿物色 + 印章 + 汉字字符画 |
| 静态界面 | 气象动效——雾飘、雨落、霜结、雪降、露凝、晴升 |

核心意象：**织机**（六梭交织）+ **气象台**（六种天气）+ **水墨**（宣纸/矿物/印章/诗）。

### 1.1 纸与墨（基底色板）

```
--paper       #f8f4ec   宣纸
--paper-warm  #f3ede2   暖宣纸（输入区）
--ink-deep    #1a1614   焦墨（正文）
--ink-mid     #3d3833   浓墨
--ink-light   #8c8680   淡墨（次要信息）
--ink-faint   #c4bfb8   飞白（分隔线/占位）
```

矿物色派生：每色提供 `solid` / `soft`(12% alpha) / `seal`(印章) 三档。

### 1.2 字体

- 中文/标题：**Noto Serif SC**（300/400/600/700）——书法骨架。
- 西文/正文：**Noto Serif**（含斜体，用于诗句）。
- 终端：等宽回退，但用真彩矿物色补回身份。

## 2. 三端一致性

设计 token 一处定义，三端各自表达同一意象。

### 2.1 Web（水墨气象台）— 现状强项，做工程化

保留并强化：宣纸纹理 `#paper-grain`、顶部水墨山影、`#ambient-layer` 气象粒子、`#kanji-decoration` 印章、侧栏诗句、切灵换主题。

工程化改进（见优化计划 P7.3）：
- 内联 HTML 抽到 `src/web/ui/{index.html,styles.css,app.js}`，token 由 `theme.ts` 注入。
- **真流式**：工具调用呈现为"气象事件"卡片（如 `web_search` → 一道雨痕划过、`apply_patch` → 一枚朱印落下）。
- 消息气泡的笔触边框按 Agent `msgStyle` 区分（已有雏形，统一到 token）。

### 2.2 TUI（全屏「立轴」）— 已落地 `src/cli/loom.ts`

`sky` 在真实终端默认进入全屏立轴（`--classic` / 管道 / 窄终端回退到线性 `tui.ts`）。

```
┌─ 天空织机 Skyloom ─────────────────────────────── 霧 ─┐  ← 印章随灵变色
│ ≋      ≋    气象粒子（随灵的动势）      ≋             │  ← 天幕 2 行
│ ▁▂▃▅▃▂▁▁▂▄▂▁  远山随会话渐渐生长  ▁▁▂▃▂▁▁▂▄▃▂▁▁     │
│ ● 霧 fog ✓1   │  流式正文，墨迹晕染入场 ▍             │  ← 矿物色侧栏 │ 视口
│ ⸽ 雨 rain     │  ✦ 织谱 · 3 梭   ← /task 多灵动态     │
│ · 霜 frost    │  ✓ ① 霧 调研 (3.2s) · ⸽ ② 雨 起草 ←① │
│ ╌╌╌╌╌╌╌╌╌╌╌╌╌ │    ⸙ web_search 检索中…               │
│ 山色有无中    │                                       │
├ ≋ 织造 1/3 ·· ───────────── gpt-4o · $0.02 · ▰▰▱▱▱ 41% ┤
│ ≋ ❯ █            ← 命令面板浮层在视口尾行             │
└─ /help 命令 · Tab 补全 · PgUp 回看 ───────────────────┘
```

架构（为何这版不闪烁、不打架）：
- 流式 token 只进**虚拟块缓冲**；每帧在内存合成整屏，差量重绘仅刷脏行。
- 宽度计算全部走 `tui.ts` 的 CJK 感知函数，中英混排/输入光标不再错位。
- 动效时钟是唯一写者：按键与流事件只改状态，由帧定时器统一落屏。

意象：
- 流式渲染 token 逐字「晕染」（淡墨→焦墨「干透」），工具调用单行天气符号脉冲。
- **多灵织机**：`/task` 编排时，运行中的灵化作梭子拖矿物色丝线在天幕穿行；
  视口内「织谱」原位更新每梭状态/依赖/耗时；左栏亮起脉冲徽记与 ✓/✗ 战绩。
- 远山剪影用 `▁▂▃▄▅` 按会话长度生长——时间感。

### 2.3 CLI（线性）— 真彩 + 印章

- 真彩矿物色替换当前 `chalk.cyan` 单色。
- 欢迎横幅：六灵以各自矿物色 + 天气符号排布，当前灵加印章方框。
- Agent 切换：打印一枚朱砂印章（`▣ 霜`）作过场。
- 流式：复用 `chatStream`，`reasoning` 以淡墨斜体、正文焦墨、工具状态淡墨脉冲。

## 3. 标志性元素（辨识度锚点）

1. **印章 (Seal)**：当前 Agent 的汉字以朱砂方框印章呈现——出现在 Web 角落、TUI 标题栏、CLI 切换过场、README badge。是品牌第一记忆点。
2. **气象粒子**：六种天气动效是 Web 的灵魂，未来可做成 TUI 的字符粒子（ASCII 雨/雪）。
3. **诗句**：每灵一句古诗，出现在侧栏/欢迎屏/空状态，是"克制的繁复"的体现。
4. **天气符号集**：`≋ ⸽ ✱ ❉ ∘ ☼`——跨端统一的 Agent 字形。

## 5. 设计 token 落地（建议代码形态）

```ts
// src/core/theme.ts — 唯一事实源
export interface AgentTheme {
  name: string; kanji: string; weather: string;
  pigment: { name: string; hex: string; rgb: [number, number, number] };
  poem: string; symbol: string; motion: string;
}
export const PALETTE = {
  paper: "#f8f4ec", paperWarm: "#f3ede2",
  inkDeep: "#1a1614", inkMid: "#3d3833", inkLight: "#8c8680", inkFaint: "#c4bfb8",
} as const;
export const AGENT_THEMES: Record<string, AgentTheme> = { /* 见 §1 表 */ };
// CLI: hex → chalk.hex(); TUI: → ANSI truecolor; Web: → CSS 变量注入
```

> 一处改色，三端同步。这是把"美观"从一次性手艺变成**可维护的系统**的关键。

## 6. 不做什么

- 不引入重型前端框架（保持零依赖 Web，宣纸般轻）。
- 不把界面"洗白"成通用 SaaS 风。
- 不让动效喧宾夺主——气象是氛围，内容是主角。
